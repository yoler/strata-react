# Harness Trace

本目录用于记录阶段性的执行过程、排查结论与临时分析结果。

适合记录：

- 某次排查的关键步骤
- 某个问题的阶段性定位结果
- 某轮重构的迁移清单
- 某次验证失败的原因与结论

不适合记录：

- 长期稳定的架构规则
- 正式架构决策
- 通用经验总结

这些内容应分别放到：

- `AGENT.md`
- `docs/ARCHITECTURE.md`
- `docs/ADR/`
- `harness/memory/`

建议命名方式：

- `trace-2026-04-08-*.md`
- `debug-*.md`
- `migration-run-*.md`

写作原则：

1. 记录足够支持后续接手，但不要写流水账
2. 明确结论、影响范围、下一步建议
3. 阶段结束后可归档或提炼到 memory / ADR
