# Harness Memory

本目录用于沉淀“正式架构规则之外”的经验信息。

适合记录：

- 技术债位置
- 某次迁移踩过的坑
- 某个模块的历史包袱
- 某个接口或依赖的特殊行为
- 对后续 AI Agent / 工程师有帮助的经验提醒

不适合记录：

- 正式架构规则
- 目录职责定义
- 已经确定的长期结构决策

这些内容应分别放到：

- `AGENT.md`
- `docs/ARCHITECTURE.md`
- `docs/ADR/`

建议命名方式：

- `feedback-*.md`
- `tech-debt-*.md`
- `migration-*.md`
- `pitfall-*.md`

写作原则：

1. 先写观察，再写影响
2. 能定位到目录或模块就不要只写抽象描述
3. 如果问题已上升为正式规则，改写为 ADR 或更新架构文档
